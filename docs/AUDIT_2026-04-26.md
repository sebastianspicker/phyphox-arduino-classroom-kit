# Deep Repository Audit — 2026-04-26

## Scope

This audit covers repository architecture, script orchestration, validation and security checks, and cross-surface consistency between firmware, phyphox sources, and generated artifacts.

Audited paths:

- `arduino/phyphox_ble_sense/`
- `src/phyphox/` and `experiments/`
- `scripts/`
- `tools/`
- `tests/`
- top-level build and config surfaces (`Makefile`, `pyproject.toml`, docs)

## How the repository works together

### 1) Authoring and generation

- Authoring source lives in `src/phyphox/*.phyphox.xml`.
- Shared fragments are included from `src/phyphox/includes/*.xml` via XInclude.
- `scripts/build-phyphox.sh` expands includes with `xmllint --xinclude`, then normalizes output through `tools/postprocess_phyphox_xml.py`.
- Generated files are committed to `experiments/*.phyphox` as import-ready artifacts.

### 2) Runtime contract

- Firmware in `arduino/phyphox_ble_sense/phyphox_ble_sense.ino` advertises BLE and streams fixed payload offsets.
- Core phyphox experiments write a mode value to config characteristic and read data characteristic.
- `tools/validate_phyphox.py` checks UUID/mode consistency across constants, firmware, source XML, and generated experiment files.

### 3) Guardrails and CI

- `scripts/validate-xml.sh` checks raw XML validity and phyphox plausibility for generated + expanded source forms.
- `scripts/check-generated-clean.sh` verifies generated artifacts match source-of-truth XML.
- `scripts/deps-scan.sh`, `scripts/secret-scan.sh`, and `scripts/sast-minimal.sh` implement the local security baseline.
- `scripts/ci-local.sh` composes lint, tests, XML validation, generation freshness, compile, and security checks.

## Findings and remediation status

### P0 (critical correctness/security)

- **No active P0 defects found** in reviewed code paths and runnable checks.

### P1 (high impact reliability/maintainability)

1. **Duplicate prerequisite checks across scripts** increased drift risk and inconsistent failure semantics.
   - **Remediation:** added shared shell helper library (`scripts/lib/common.sh`) and centralized prerequisite profiler (`scripts/check-prereqs.sh`).
   - **Status:** fixed.

2. **`ci-local` prerequisite failures were discovered late** (after partial execution in some environments).
   - **Remediation:** `scripts/ci-local.sh` now performs a single up-front prerequisite check (`ci` profile) before running any stage.
   - **Status:** fixed.

### P2 (optimization/dedup/polish)

1. **Secret scan implementation was O(patterns × files)** and repeatedly invoked `rg` per file per pattern.
   - **Remediation:** `scripts/secret-scan.sh` now performs one batched ripgrep invocation over tracked/untracked files with grouped patterns.
   - **Status:** fixed.

2. **Developer bootstrap lacked a single, explicit tool check command.**
   - **Remediation:** added `make prereqs` (with `PROFILE=<base|build|compile|security|ci>`), wired to `scripts/check-prereqs.sh`; updated docs accordingly.
   - **Status:** fixed.

3. **Pytest config included an unsupported option in this environment**, generating noise.
   - **Remediation:** removed unsupported `asyncio_default_fixture_loop_scope` setting from `pyproject.toml`.
   - **Status:** fixed.

## Remaining recommendations

1. Keep prerequisite checks mandatory in contributor workflows (`PROFILE=ci make prereqs` before full loop).
2. If desired, split CI into soft/hard stages for environments without Arduino toolchain (policy choice, not a correctness fix).
3. Preserve strict generated parity enforcement (`check-generated-clean.sh`) as a release gate.
